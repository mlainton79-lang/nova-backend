## Verdict (verbatim)

> The new self-check can misreport completed task activity and can schedule duplicate daily self-check tasks across restarts. These are functional issues in the added feature that should be fixed before treating the patch as correct.

## Findings (verbatim)

- **[P2] Count the queue's actual done status** — `/mnt/sdcard/Download/nova-backend/app/core/self_check.py:90-90`
  When tasks complete normally, `task_queue._execute_task` writes `status = 'done'`, not `completed`, so any real completed work in the last 24 hours is reported as `0 done`. This makes the new self-check's task summary misleading even when the worker is healthy; count `done` here or align the status names.

- **[P2] Dedupe against already scheduled self-checks** — `/mnt/sdcard/Download/nova-backend/app/core/self_check.py:218-220`
  If the service restarts or redeploys more than 6 hours after a self-check was queued but before it has run, this query no longer sees the existing future task and queues another one for the same target time. That can produce duplicate morning push notifications; dedupe on active `self_check` tasks for the scheduled date/time instead of only `created_at > 6 hours`.
